# Requirements Analysis

Implemented:

- Hard receive safety constraints.
- Pass line obstruction checks.
- Opponent interception time prediction.
- Dynamic receive point prediction from teammate velocity.
- Fixed-point pass line validation.
- Fallback when no safe pass exists.
- Explainable per-candidate scores.
- Unit tests and experiment outputs.

Not verified because of missing official Runner configs:

- Webots simulation time progression through Runner.
- SDK state read from simulated robot.
- T1 stand, walk, turn, and stop in Webots.
