# 需求规格说明书 (SRS)

> 项目：Booster T1 多机器人足球协同决策系统（题目四）  
> 版本：v1.0

## 1. 项目背景

基于 Booster T1 机器人平台，采用软件工程方法实现多机器人足球协同决策系统，支持传球、带球、跑位、卡位和简单攻防决策。

## 2. 用户角色

| 角色 | 描述 |
|------|------|
| 开发人员 | 实现策略模块、决策引擎、测试 |
| 演示人员 | 运行仿真、录制视频、导出日志 |
| 评审教师 | 验收功能、查看文档和测试报告 |

## 3. 功能需求

### FR-01 世界状态读取
系统应基于统一 `WorldState` 读取球、队友、对手和球门位置。

### FR-02 传球
两个机器人应能完成一次有效传球（判断接球区域、执行 PASS、队友移动到接球点）。

### FR-03 带球推进
持球机器人应能向球门方向带球推进。

### FR-04 射门决策
在合适距离和角度时执行射门。

### FR-05 跑位与卡位
非持球机器人执行支援跑位；检测到威胁时执行防守卡位。

### FR-06 决策状态机
实现 CHASE → DRIBBLE → PASS → SHOOT → BLOCK 状态切换。

### FR-07 角色分配
自动分配 ball_carrier / supporter / defender 角色。

### FR-08 决策日志
完整输出决策过程，支持 CSV 导出。

## 4. 非功能需求

| 编号 | 需求 |
|------|------|
| NFR-01 | Python 3.8+，标准库为主 |
| NFR-02 | Mock 模式可独立运行，无需实机 |
| NFR-03 | 测试覆盖率 ≥ 15 条用例 |
| NFR-04 | 代码模块化，公共接口可冻结扩展 |

## 5. 验收标准

- [x] 两个机器人基于统一 WorldState 完成传球或带球推进
- [x] 系统输出完整决策过程日志
- [x] `python main.py` 可运行
- [x] `--scenario pass/shoot/threat` 可演示
- [x] `--export-csv` 可导出日志
- [x] `python -m pytest -q` 全部通过（57/57）
- [ ] 演示视频（待录制）
- [ ] 答辩 PPT（待准备）

## 6. 约束与假设

- 当前阶段使用内置 2D Mock 仿真，不接 Booster 实机
- Windows 环境需设置 `$env:PYTHONUTF8="1"` 避免 emoji 编码错误
- Ubuntu/ROS2/WSL 为进阶扩展，非最低验收前置条件
